## アプリ概要
これまで個人開発で制作してきたアプリケーションをまとめた、
**プロダクト集合型のポートフォリオサイト**です。

単に成果物を並べるのではなく、

- 私自身がどのような人間であるのか
- なぜこのアプリを作ろうと思ったのか
- どんな人の、どんな困りごとを想定したのか
- どのように設計・実装したのか

といった **思考の過程ごと伝える** ことを目的としています。

## 開発背景
個人開発を続ける中で、

- アプリごとにGitHubが分かれていて全体像が伝わりにくい
- READMEだけでは「背景」や「想い」まで書ききれない
- 技術だけでなく、考え方も含めて知ってもらいたい

と感じるようになりました。

そこで、
**プロダクト単体ではなく「作り手としての軸」をまとめて伝えられる場所**として、
このポートフォリオサイトを制作しました。

## 制作への想い
私は、プロダクトは
**人の「負」を少し軽くするための道具** だと考えています。

不安、迷い、悩み、言語化できない気持ち。
そういったものを直接的に解決したり、
「考えるきっかけ」や「整理する視点」を提供できるアプリを作りたい。

このポートフォリオには、
そんな想いで向き合ってきたプロダクトと、その背景を詰め込みました。

## 想定ユーザー（ペルソナ）
- エンジニア採用担当者・面接官の方
- 実際に一緒に開発する可能性のあるエンジニア
- 個人開発や学習の過程に興味のある方
- そして、自分自身（振り返り・成長ログとして）

## MVP 機能
- トップページ
  - 自己紹介
  - 開発スタンス・価値観の言語化
- プロダクト一覧
  - 個人開発・卒業制作を一覧表示
- プロダクト詳細ページ
  - アプリ概要
  - 開発背景（なぜ作ったか）
  - 使用技術
  - 工夫した点
  - 今後の展望
- GitHub / 本番URL へのリンク
- レスポンシブ対応（PC / スマホ）

## 本リリース
- 技術記事・振り返り記事の掲載
- プロダクト検索・フィルタ機能
- ダークモード対応
- OGP・SNSシェア最適化
- アクセシビリティ配慮
- アナリティクス導入

## 技術スタック（MVP 想定）

| カテゴリ | 技術 | 選定理由 |
| --- | --- | --- |
| Backend / Frontend | Ruby on Rails | MVC構造で設計意図を明確に表現でき、UIからDBまで一貫して実装できるため |
| View | ERB | Rails標準のテンプレートエンジンで、学習コストを抑えつつ構造を明示できるため |
| Styling | Tailwind CSS | コンポーネント設計とデザイン調整のしやすさを両立できるため |
| JavaScript | Hotwire（Turbo / Stimulus） | SPAにせずとも体験を向上でき、Railsとの親和性が高いため |
| Database | PostgreSQL | 実務での利用頻度が高く、将来的な拡張にも対応しやすいため |
| Image Upload | ActiveStorage | Rails標準機能で画像管理をシンプルに実装できるため |
| Infrastructure | Docker / Docker Compose | 開発環境を再現可能にし、環境差異を減らすため |
| Version Control | Git / GitHub | 開発履歴・設計意図を可視化するため |

## 技術的配慮と工夫
- 各プロダクトで **「背景 → 課題 → 解決」** が自然に伝わる構成
- 技術名の列挙ではなく、**なぜその技術を選んだか** を明記
- 読み手が迷わないよう、1ページあたりの情報量を調整
- スマートフォン閲覧時も読みやすいレイアウト

## 差別化ポイント
- 複数のアプリを **思想・テーマ軸** で整理している点
- 技術力だけでなく、**課題設定や思考プロセス** まで見せている点
- 継続的に更新される前提の設計（成長過程も含めたポートフォリオ）

## このアプリが提供する価値
- 採用側にとって
　→ 技術スキルだけでなく、**考え方や価値観** が伝わる  
- 自分にとって
　→ 学習と個人開発を、一本のストーリーとして残せる
